- **分层架构**：采用 `CLI` (rdx/cli.py) -> `Daemon` (rdx/daemon/server.py) -> `Worker` (rdx/runtime_worker.py) -> `Core Engine` (rdx/core/engine.py) 的四层调用链。CLI 作为轻量前端，通过 Windows 命名管道与 Daemon 通信；Daemon 负责生命周期管理与状态持久化；Worker 为独立子进程，承载重型 RenderDoc 运行时以避免主进程阻塞。
- **核心引擎与注册表**：`CoreEngine` 统一处理操作执行、异常映射与标准化响应信封（canonical envelope）。`OperationRegistry` 动态注册来自 `spec/tool_catalog.json` 的 196+ 个工具操作，实现逻辑解耦。
- **会话与上下文模型**：通过 `SessionManager` 和 `ContextSnapshot` 管理多隔离上下文（Context），支持本地与远程（Android/Remote）后端切换。状态通过 `runtime_state` 模块在磁盘与内存间同步，确保跨请求的状态一致性。
- **数据契约**：使用 `pydantic` 定义严格的领域模型（`rdx/models.py`），涵盖 Capture、Session、Shader Patch、Experiment 等核心实体，确保工具间数据交换的类型安全。
